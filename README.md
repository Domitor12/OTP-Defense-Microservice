```text
 ┌────────────────────────────────────────────────────────────────────┐
 │                                                                    │
 │   OTP Defense is a production-grade microservice that protects     │
 │   your authentication systems from:                                │
 │                                                                    │
 │   🤖 Automated OTP Bots        📞 Vishing Attacks                 │
 │   🔑 Credential Stuffing       💣 SMS/Call Bombing                │
 │   🌐 Malicious IP Addresses    ⚡ Rate Limit Abuse                 │
 │                                                                    │
 └────────────────────────────────────────────────────────────────────┘
```

This system evaluates every OTP request in **< 10ms** using a multi-layered defense pipeline:

1. **IP Reputation Check** - Blocks known malicious IPs instantly
2. **Failed Login Velocity** - Detects credential stuffing attacks
3. **OTP Rate Limiting** - Prevents SMS/call bombing
4. **Audit Logging** - Immutable records for compliance & forensics

## 📦 𝙸𝚗𝚜𝚝𝚊𝚕𝚕𝚊𝚝𝚒𝚘𝚗

### Prerequisites

- Docker & Docker Compose
- Python 3.12+ (for local development)
- PostgreSQL 16+ (if running without Docker)
- Redis 7+ (if running without Docker)

### Local Development Setup

```bash
# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Set up environment variables
cp .env.example .env
# Edit .env with local database credentials

# Run database migrations
alembic upgrade head

# Start the service
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Docker Deployment

```bash
# Production build
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

# With specific environment file
docker compose --env-file .env.production up -d

# View logs
docker compose logs -f api
```

All configuration is managed through environment variables. See `.env.example` for the complete list.

### Critical Settings

```bash
# ─── Database ─────────────────────────────────────
POSTGRES_USER=otp_defense
POSTGRES_PASSWORD=<strong-password-here>
POSTGRES_DB=otp_defense_db
POSTGRES_HOST=postgres
POSTGRES_PORT=5432

# ─── Redis ────────────────────────────────────────
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=<strong-password-here>

# ─── Rate Limits ──────────────────────────────────
FAILED_LOGIN_WINDOW_SECONDS=300      # 5 minutes
FAILED_LOGIN_MAX_ATTEMPTS=3          # Block after 3 failures
OTP_REQUEST_WINDOW_SECONDS=600       # 10 minutes
OTP_REQUEST_MAX_ATTEMPTS=2           # Challenge after 2 requests

# ─── Security ─────────────────────────────────────
API_AUTH_TOKEN=<generate-strong-token>
```

## 🔒 𝚂𝚎𝚌𝚞𝚛𝚒𝚝𝚢 𝙲𝚘𝚗𝚜𝚒𝚍𝚎𝚛𝚊𝚝𝚒𝚘𝚗𝚜

### Implemented Security Measures

✅ **Authentication**: Bearer token for service-to-service calls  
✅ **Network Isolation**: Backend services on internal Docker network  
✅ **Input Validation**: Strict Pydantic schemas with regex validation  
✅ **SQL Injection Prevention**: SQLAlchemy ORM with parameterized queries  
✅ **Rate Limiting**: Atomic Redis Lua scripts prevent race conditions  
✅ **Audit Logging**: Immutable records for all decisions  
✅ **Graceful Degradation**: Service continues if Redis fails  

### Threat Model

```text
 ┌─────────────────────────────────────────────────────────────┐

 │  💣 SMS Bombing          → Per-phone rate limits            │
 │  🌐 Malicious IPs        → Threat intel integration         │

 │  🔓 Data Breach          → Network isolation + Encryption   │
```
